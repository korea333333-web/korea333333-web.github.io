Grok 4.5, 코딩에서 엑셀·PPT까지: 업무 AI 경쟁이 넓어졌다

SpaceXAI가 7월 8일 공개한 Grok 4.5는 코딩 전용 모델에 머물지 않는다. Cursor와 함께 훈련한 코딩 능력을 앞세우면서 Grok Build와 Microsoft Office 연결을 통해 Excel 모델, PowerPoint 자료, Word 문서까지 다룬다. 개발자 도구와 사무 업무 에이전트의 경계가 빠르게 흐려지는 출시다.

핵심 요약

최고 성능 하나보다 속도, 가격, 업무 도구 연결을 함께 판다

Grok 4.5는 Grok Build의 기본 모델이며 Cursor 전 요금제와 SpaceXAI API에서 제공된다. API 가격은 입력 100만 토큰당 2달러, 출력 100만 토큰당 6달러다. 회사는 초당 80토큰과 높은 토큰 효율을 강조하지만, 공개한 코딩 평가 다섯 개 모두에서 1위를 한 것은 아니다. 경쟁력의 중심은 최고점 하나보다 빠른 응답과 비용, 실제 도구 연결의 조합에 있다.

코딩 성능은 어디쯤인가

SpaceXAI가 공개한 결과만 보면 Grok 4.5는 일부 평가에서 선두권이지만 모든 항목의 1위는 아니다. SWE Marathon에서는 비교 모델 중 가장 높은 29.0%를 기록했다. 반면 DeepSWE 1.0과 1.1, Terminal Bench 2.1, SWE Bench Pro에서는 다른 모델이 더 높은 점수를 냈다.

이 숫자는 각 모델의 실제 체감 품질을 그대로 순위화하지 않는다. 평가마다 실행 도구와 추론 설정이 다르고, 경쟁 모델의 수치는 해당 개발사의 시스템 카드나 공개 순위표에서 가져왔다. TechCrunch도 Grok 4.5의 결과가 경쟁력은 보여주지만 최상위에는 조금 못 미친다고 설명했고, Axios는 최신 대형 경쟁 모델보다 뒤지는 항목이 있다고 짚었다.

토큰 효율과 속도는 회사 주장으로 봐야 한다

SpaceXAI는 Grok 4.5가 초당 80토큰으로 제공되고, SWE Bench Pro 한 작업에서 평균 15,954개의 출력 토큰을 사용해 Opus 4.8 max의 67,020개보다 4.2배 적었다고 밝혔다. 비용에 민감한 코딩 에이전트에는 의미 있는 차이다. 다만 한 평가의 평균 토큰 수가 모든 업무의 비용 절감으로 이어지는 것은 아니다. 실제 비용은 재시도 횟수, 도구 호출, 긴 문맥, 사람이 수정하는 시간까지 포함해 봐야 한다.

ChatGPT Work와 무엇이 다른가

두 제품은 겹치는 부분이 있지만 출발점이 다르다. Grok 4.5 발표는 모델과 API, Cursor 코딩, Grok Build 및 Office 추가 기능을 중심으로 한다. ChatGPT Work는 브라우저, 파일, 앱, 연결 도구, 반복 작업을 한 작업 공간에서 실행하는 방식이 중심이다. 하나는 빠르고 저렴한 모델을 여러 도구에 넣는 전략에 가깝고, 다른 하나는 사용자의 업무 절차 전체를 실행 환경으로 묶는 전략에 가깝다.

한국 개발자와 기업이 볼 부분

개발팀은 공개 점수보다 실제 저장소에서의 성공률과 수정 비용을 먼저 비교해야 한다. 특히 보안이 중요한 코드, 사내 문서, 재무 모델을 연결할 때는 어떤 데이터가 모델 제공사로 전송되고 어느 도구가 파일을 수정할 수 있는지 확인해야 한다. 오피스 업무에서는 멋진 첫 결과보다 수식 오류, 출처 누락, 슬라이드 수정 가능성, 사내 템플릿 유지가 더 중요하다.

앞으로 볼 포인트

  • 독립 개발자 평가에서 공개 코딩 점수와 비슷한 성능이 재현되는지
  • Excel 수식, PowerPoint 기본 도형, Word 서식이 실제 파일에서 얼마나 안정적으로 유지되는지
  • Cursor와 Grok Build 사용량을 포함한 전체 비용이 경쟁 제품보다 낮은지
  • 기업용 데이터 보호, 감사 기록, 도구 권한 관리가 어떻게 제공되는지
