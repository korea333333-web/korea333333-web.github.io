Grok 4.5, 코딩에서 엑셀·PPT까지: 업무 AI 경쟁이 넓어졌다

7월 8일 언론 보도로 먼저 알려지고 SpaceXAI가 7월 16일 공식 소개 페이지를 게시한 Grok 4.5는 코딩 전용 모델에 머물지 않는다. Cursor와 함께 훈련한 코딩 능력을 앞세우면서 Grok Build와 Microsoft Office 연결을 통해 Excel 모델, PowerPoint 자료, Word 문서까지 다룬다. 개발자 도구와 사무 업무 에이전트의 경계가 빠르게 흐려지는 출시다.

핵심 요약

최고 성능 하나보다 속도, 가격, 업무 도구 연결을 함께 판다

Grok 4.5는 Grok Build의 기본 모델이며 Cursor 전 요금제와 SpaceXAI API에서 제공된다. API 가격은 입력 100만 토큰당 2달러, 출력 100만 토큰당 6달러다. 회사는 초당 80토큰과 높은 토큰 효율을 강조하지만, 공개한 코딩 평가 다섯 개 모두에서 1위를 한 것은 아니다. 경쟁력의 중심은 최고점 하나보다 빠른 응답과 비용, 실제 도구 연결의 조합에 있다.

7월 8일 보도와 7월 16일 공식 페이지를 구분해야 한다

Axios와 TechCrunch는 7월 8일 출시 소식과 가격, 초기 배포 범위를 보도했다. 현재 SpaceXAI 공식 소개 페이지에는 7월 16일이라는 날짜가 표시된다. 따라서 이 글에서는 7월 8일을 최초 공개 보도 시점, 7월 16일을 회사가 상세 기술 자료를 게시하거나 갱신한 시점으로 구분한다.

초기 보도와 현재 공식 페이지 사이에는 배포 설명도 조금 다르다. Axios는 당시 유럽연합에서 아직 이용할 수 없다고 전했고, 공식 페이지는 Grok Build·Cursor 전 요금제·API 콘솔 제공을 명시한다. 지역별 제공 여부는 계정과 계약 시점에 따라 바뀔 수 있으므로, 실제 도입 전에는 콘솔에서 사용 가능 지역과 기업 약관을 다시 확인해야 한다.

이 구분은 사소한 날짜 정리가 아니다. 빠르게 바뀌는 모델 출시에서는 최초 기사, 회사 블로그, 실제 API 문서가 서로 다른 시점의 상태를 담을 수 있다. 구매나 개발 일정을 정할 때는 기사 제목보다 현재 콘솔의 모델 ID, 요금표, 지역 제한, 데이터 처리 약관을 기준으로 삼는 편이 안전하다.

모델 뒤에는 대규모 학습과 에이전트 훈련이 있다

SpaceXAI는 Grok 4.5를 수만 대의 NVIDIA GB300 GPU에서 훈련했다고 설명한다. 데이터 중복 제거와 품질 점수화, 분야별 선별을 거쳤고 코딩·과학·공학·수학 데이터를 폭넓게 사용했다는 것이 회사 설명이다. 다만 구체적인 파라미터 수, 학습 토큰 수, 데이터 구성 비율은 공개하지 않았다.

강화학습은 수십만 개의 과제를 대상으로 했으며, 여러 단계의 소프트웨어 엔지니어링 작업과 장시간 에이전트 실행을 강조했다. 이는 한 번의 답변을 잘 쓰는 챗봇보다 저장소를 읽고 터미널을 조작하며 실패 후 다시 시도하는 시스템을 겨냥했다는 뜻이다. 모델 점수만큼 도구 실행 환경과 오류 복구 방식이 실제 성능을 좌우한다.

반대로 회사가 공개한 학습 규모가 안전성이나 정확성을 자동으로 보장하지는 않는다. 데이터 출처, 보안 평가, 위험한 도구 호출에 대한 차단 방식은 별도의 검토 대상이다. 기업 사용자는 모델 카드와 API 약관뿐 아니라 로그 보존, 학습 데이터 재사용 여부, 하위 처리업체 목록까지 계약 문서에서 확인할 필요가 있다.

코딩 성능은 어디쯤인가

SpaceXAI가 공개한 결과만 보면 Grok 4.5는 일부 평가에서 선두권이지만 모든 항목의 1위는 아니다. SWE Marathon에서는 비교 모델 중 가장 높은 29.0%를 기록했다. 반면 DeepSWE 1.0과 1.1, Terminal Bench 2.1, SWE Bench Pro에서는 다른 모델이 더 높은 점수를 냈다.

이 숫자는 각 모델의 실제 체감 품질을 그대로 순위화하지 않는다. 평가마다 실행 도구와 추론 설정이 다르고, 경쟁 모델의 수치는 해당 개발사의 시스템 카드나 공개 순위표에서 가져왔다. TechCrunch도 Grok 4.5의 결과가 경쟁력은 보여주지만 최상위에는 조금 못 미친다고 설명했고, Axios는 최신 대형 경쟁 모델보다 뒤지는 항목이 있다고 짚었다.

토큰 효율과 속도는 회사 주장으로 봐야 한다

SpaceXAI는 Grok 4.5가 초당 80토큰으로 제공되고, SWE Bench Pro 한 작업에서 평균 15,954개의 출력 토큰을 사용해 Opus 4.8 max의 67,020개보다 4.2배 적었다고 밝혔다. 비용에 민감한 코딩 에이전트에는 의미 있는 차이다. 다만 한 평가의 평균 토큰 수가 모든 업무의 비용 절감으로 이어지는 것은 아니다. 실제 비용은 재시도 횟수, 도구 호출, 긴 문맥, 사람이 수정하는 시간까지 포함해 봐야 한다.

명목 가격으로만 보면 출력 100만 토큰당 6달러는 장시간 코딩 작업을 많이 돌리는 팀에 매력적이다. 그러나 에이전트는 답변 한 번으로 끝나지 않고 파일 읽기, 검색, 테스트, 수정 과정을 반복한다. 같은 업무를 몇 번의 시도로 끝내는지까지 측정해야 실제 작업당 비용을 비교할 수 있다.

예를 들어 월간 토큰비가 낮아도 잘못된 코드 검토와 회귀 테스트에 개발자 시간이 더 들면 총비용은 오히려 커진다. 반대로 빠른 응답과 적은 출력 토큰이 실제 성공률과 함께 재현된다면, 대량의 사내 자동화나 병렬 코드 검토에서 가격 차이가 크게 누적될 수 있다. 핵심 지표는 토큰 단가가 아니라 성공한 작업 한 건당 비용이다.

Excel과 PowerPoint 지원이 중요한 이유

공식 예시는 Grok Build가 웹 조사를 거쳐 여러 시트와 수식이 있는 Excel 모델을 만들고, 나중에 검토할 메모까지 남기는 흐름을 보여준다. PowerPoint에서는 기본 도형으로 다이어그램을 만들고 Word에서는 문서를 작성한다고 설명한다. 결과를 이미지로만 내놓는 것이 아니라 수정 가능한 오피스 객체로 남기려는 접근이다.

업무 현장에서는 이 차이가 크다. 재무팀은 셀의 근거와 수식을 추적해야 하고, 기획팀은 회사 템플릿 안에서 도형과 문구를 다시 편집해야 한다. 보기 좋은 결과보다 계산 과정과 편집 가능성을 유지하는지가 도입 여부를 가른다.

아직 공식 데모만으로 복잡한 재무 모델이나 대규모 프레젠테이션의 안정성을 판단할 수는 없다. 숨겨진 시트 참조, 순환 수식, 통화·날짜 서식, 폰트 대체, 마스터 슬라이드 훼손처럼 실제 파일에서 자주 생기는 문제를 별도 표본으로 시험해야 한다. 민감한 문서를 연결할 때는 추가 기능이 요구하는 권한 범위도 반드시 점검해야 한다.

ChatGPT Work와 무엇이 다른가

두 제품은 겹치는 부분이 있지만 출발점이 다르다. Grok 4.5 발표는 모델과 API, Cursor 코딩, Grok Build 및 Office 추가 기능을 중심으로 한다. ChatGPT Work는 브라우저, 파일, 앱, 연결 도구, 반복 작업을 한 작업 공간에서 실행하는 방식이 중심이다. 하나는 빠르고 저렴한 모델을 여러 도구에 넣는 전략에 가깝고, 다른 하나는 사용자의 업무 절차 전체를 실행 환경으로 묶는 전략에 가깝다.

한국 개발자와 기업이 볼 부분

개발팀은 공개 점수보다 실제 저장소에서의 성공률과 수정 비용을 먼저 비교해야 한다. 특히 보안이 중요한 코드, 사내 문서, 재무 모델을 연결할 때는 어떤 데이터가 모델 제공사로 전송되고 어느 도구가 파일을 수정할 수 있는지 확인해야 한다. 오피스 업무에서는 멋진 첫 결과보다 수식 오류, 출처 누락, 슬라이드 수정 가능성, 사내 템플릿 유지가 더 중요하다.

가장 현실적인 비교 방법은 실제 업무 20~30개를 고정 표본으로 만드는 것이다. 버그 수정, 테스트 작성, 오래된 코드 설명, 다중 시트 예산 모델, 기존 템플릿 기반 보고서처럼 팀이 반복하는 작업을 넣고 완성률·재시도 횟수·사람의 수정 시간·총토큰을 함께 기록하면 된다.

파일을 직접 수정하는 에이전트에는 최소 권한 원칙도 필요하다. 운영 저장소나 원본 재무 파일에 곧바로 연결하지 말고 복사본과 격리된 브랜치에서 시작해 변경 내역을 사람이 승인하도록 해야 한다. 감사 로그와 되돌리기 절차가 없는 자동화는 모델 성능이 좋아도 기업 도구로 쓰기 어렵다.

내가 보는 핵심

Grok 4.5의 의미는 또 하나의 코딩 벤치마크 1위 경쟁보다 모델 공급사가 개발 도구와 오피스 문서를 하나의 업무 시장으로 보기 시작했다는 데 있다. Cursor에서 코드를 만들고 Grok Build에서 조사하며 Excel과 PowerPoint 결과물까지 이어지는 흐름은 개발자용 AI와 일반 사무용 AI의 구분을 약하게 만든다.

다만 지금 공개된 근거의 상당 부분은 SpaceXAI 자체 평가와 제품 데모다. 가격과 속도는 눈에 띄지만, 독립 평가와 실제 파일 호환성, 기업 데이터 보호가 확인돼야 강점이 완성된다. 지금 단계에서 가장 좋은 태도는 최고 모델을 선언하는 것이 아니라 작은 실무 표본으로 성공한 작업당 비용을 직접 재는 것이다.

앞으로 볼 포인트

  • 독립 개발자 평가에서 공개 코딩 점수와 비슷한 성능이 재현되는지
  • Excel 수식, PowerPoint 기본 도형, Word 서식이 실제 파일에서 얼마나 안정적으로 유지되는지
  • Cursor와 Grok Build 사용량을 포함한 전체 비용이 경쟁 제품보다 낮은지
  • 기업용 데이터 보호, 감사 기록, 도구 권한 관리가 어떻게 제공되는지
